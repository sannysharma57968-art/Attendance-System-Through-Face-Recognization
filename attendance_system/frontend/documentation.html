Project Documentation

1. System Architecture & Components

The system follows a modular architecture, consisting of the following key components:

Backend Components (Python/FastAPI)

  • main.py: The central entry point. Orchestrates the API endpoints and video streaming.
  • face_engine.py: Handles face detection (HOG) and recognition (deep learning).
  • camera_manager.py: Threaded non-blocking video capture with auto-reconnect.
  • database.py: SQLite management for persistent attendance logs.

Frontend Components (Vanilla JS/CSS)

  • index.html / admin.html: Responsive dashboard and administration portals.
  • app.js: Handles API calls, real-time UI updates, and data visualization.

2. System Work Flow (Fork Flow)

Step 1: Frame Capture

The Camera Thread captures raw frames from the webcam in real-time.

Step 2: Pre-processing

Frames are downscaled to 25% resolution to minimize CPU consumption during detection.

Step 3: Recognition

Face features are extracted and compared against the registered database encodings.

Step 4: Stability Check

A face must be identified for 3 consecutive frames to avoid flickering or false positives.

Step 5: Record Entry

The system verifies if attendance is already marked; if not, it logs the time in the database.

3. Key Technologies

  • FastAPI: High-speed asynchronous web framework.
  • Face Recognition: Dlib-based state-of-the-art face identification.
  • OpenCV: Real-time computer vision and image processing.
  • SQLite: Lightweight and efficient data storage.
